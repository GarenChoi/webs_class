식별자/키워드

키워드

자바스크립트가 만들어질 때 정해진 특별한 의미가 있는 단어입니다. 키워드들은 식별자로 정의할 수 없습니다.

식별자

자바스크립트에서 변수나 함수의 이름이 될 수 있는 프로그래머가 선언하는 단어입니다.


식별자 명명규칙

1. 공백이 들어가면 안된다.
2. 특수문자는_와 $만 가능하다.
3. 숫자로 시작하면 안된다.
4. 키워드를 사용하면 안된다.


식별자 관례

1. 한자, 한글을 사용할 수 있으나 영어를 사용하는게 관례이다.
2. 변수, 인스턴스, 함수, 메서드의 이름은 소문자로 시작한다.
3. 생성자 함수의 이름은 대문자로 시작한다
4. 여러단어로 이루어진 식별자는 각 단어의 첫 글자를 대문자로 한다.


식별자의 종류

식별자는 크게 4가지(변수, 속성, 함수, 메서드) 종류로 나눌 수 있습니다.

구분 단독으로 사용 다른 식별자와 사용
식별자 뒤에 괄호 O 변수 속성
식별자 뒤에 괄호 X 함수 메소드

단어 조합 규칙

프로그래밍을 할 때 두개 이상의 단어를 조합해 변수나 클래스를 만드는 경우가 있습니다. 그럴 경우 공백은 사용할 수 없기 때문에 프로그래머들이 내놓은 다양한 해답 중 가장 자주 쓰이는 것들이 카멜식(Camel Case), 케밥식(Kebab Case), 파스칼식(Pascal Case), 스네이크식(Snake Case) 입니다

1. 카멜식(camelCase)

: 첫 단어는 소문자로 시작하고, 두번째 단어부터 대문자로 시작한다. 단어와 단어 사이는 붙인다.

2. 케밥식(kebab-case)

: 모든 단어가 소문자로 되어 있고, 단어와 단어 사이는 바(-)로 연결된다.

3. 파스칼식(PascalCase)

: 모든 단어가 대문자로 시작하고, 단어와 단어 사이는 붙인다.

4. 스네이크식(snake_case)

: 모든 단어가 소문자로 되어 있고, 단어와 단어 사이는 언더바(_)로 연결된다.
